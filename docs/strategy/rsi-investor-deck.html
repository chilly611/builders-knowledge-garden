BKG is wired around five feedback loops that fire silently on every query. These loops watch the gap between what the system said and what actually happened. That gap is currency.

Solid arrows: data in. Dashed arrows: feedback loops back to engine.

Loop 1: Code & Data Drift Detection

What It Does

Every jurisdiction amendment, every NEC update, every code edition becomes a delta.

Daily heartbeat job runs at 11am UTC. Pulls latest code databases from ASHRAE, IBC, IRC, NEC, IPC, IMC and local amendments. Detects which sections changed.

When the NEC moves receptacle spacing rules, we know. When a local AHJ publishes a variance, we catch it.

Sample Code Drift Detection

NEC 2023 Update Section 210.52(A) - Receptacle Spacing - OLD: "Receptacles in kitchens shall be spaced..." + NEW: "Kitchen islands/peninsulas require 2ft spacing..." Local Amendment: San Francisco - EV charging required by code update - Section 626.1 (new)

System detects, flags for review, updates knowledge index automatically.

Loop 2: Specialist Prompt Improvement

What It Does

Every wrong answer is a training signal. The system learns.

Telemetry logs every specialist call: question, answer, user feedback. Signals: thumbs_up, thumbs_down, correction, outcome_success/failure.

Synthesis engine clusters feedback by specialist + signal type, proposes concrete delta, human approves, prompt auto-deploys.

Example: Kitchen Receptacle Correction

BEFORE

Q: Kitchen countertop receptacles?

A: "Receptacles required along counters."

❌ Missing: GFCI requirement in wet locations

AFTER

Q: Kitchen countertop receptacles?

A: "All kitchen counters require receptacles spaced 2 ft apart, GFCI-protected within 6 ft of sink."

✓ Complete & accurate

22 specialists. 2,200 queries/week needed to spin this loop reliably.

Loop 3: Workflow Shape Evolution

What It Does

Every drop-off becomes a UX rewrite. If contractors abandon "Plan it out" at 40%, we change it.

Contractors flow through 7 stages: Size Up → Lock → Plan → Build → Adapt → Collect → Reflect.

We log entry, exit, user edits, time spent. High drop-off stage? Retrain copilot, reorder fields, split step.

Loop 4: Vendor Adapter Accuracy

What It Does

Every wrong price, every missed lead time becomes a retrain.

Every vendor lookup, every quote, every actual order is logged. When BKG says "this roof costs $12K" and contractor pays $11.2K, the adapter learns.

Over time, quotes inch closer to reality because we observe the ground truth on every project that reaches "Collect" stage.

Price Correction Example: Roofing in Texas

Observation Window: Q1 2025 Texas projects Sample Corrections: Asphalt shingle, 2,000 sq ft → BKG estimate: $14,000 → Actual market: $12,800 (-8.6%) → Adapter learns: apply -8% factor for TX Metal roofing, 3,000 sq ft → BKG estimate: $19,500 → Actual market: $18,700 (-4.1%) → Trend: metal has smaller variance

Loop 5: Copilot Quality

What It Does

Every latency spike, every missed citation becomes a system prompt revision.

Signals: response latency, citation accuracy, user correction patterns, edit frequency.

Synthesis engine proposes prompt tweaks: shorter context window, stricter citation constraint, fallback to "defer to human" flag.
