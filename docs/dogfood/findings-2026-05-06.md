## Bug surfaced during this verification

### B-8: AI summary gets overwritten by every subsequent copilot call

**Severity:** Annoying, not demo-blocking.
**What:** When you create a project from `/killerapp` with raw_input, the first AI orientation populates `command_center_projects.ai_summary` (first ~600 chars). That's the desired source for the banner's AI take preview. **But every subsequent `/api/v1/copilot` call against that project_id overwrites `ai_summary`** — including ones that aren't orientation. Yesterday I ran a hallucination probe via the AI fab ("What does NEC 919.7(D)(4) say…?"); today the banner across all wired workflows shows that hallucination-disclaimer response instead of the original ADU orientation.
**Fix path:** In `src/app/api/v1/copilot/route.ts` `persistProjectExchange`, only update `ai_summary` if the project's current `ai_summary` is null/empty. Or better: introduce a separate `orientation_summary` column that only the killerapp landing flow writes to, leaving `ai_summary` for the latest exchange. Either is ~10-15 min of code.
**Adding to Tier 1.5 fix-list.**
